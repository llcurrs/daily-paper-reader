## 摘要
多模态融合和多模型集成在提升3D语义分割性能中很常见。尽管性能令人印象深刻，但这些方法要么依赖辅助输入信号，要么计算成本高昂。为了在不引入不可承受成本的情况下有效提升分割性能，我们提出通过知识蒸馏将多模态模型（即点云和图像）和多个模型专家的丰富知识转移到基于点云的网络中。具体而言，我们提出了面向信息的异构蒸馏（IHD），以帮助单模态模型吸收来自多模态教师的互补知识。我们设计了面向信息的过滤（IOF）策略，从连续图像序列中选择信息丰富的图像用于多模态融合。这种做法可以提升多模态教师的性能，从而有利于学生的学习。此外，与需要单独训练每个专家的传统模型集成不同，我们提出了擅长快照蒸馏（ASD）。ASD将训练阶段自由生成的模型快照视为多个专家，这显著降低了模型集成的训练成本。对于每个专家教师，它仅在其擅长的类别上为学生提供监督。由此产生的异构与擅长快照知识蒸馏（HAS-KD）在ScanNetV2和S3DIS数据集上取得了最先进的结果。HAS-KD可以无缝集成到当前的3D分割算法中，并在不引入额外推理负担的情况下带来可观的收益。代码将在发表后公开。

## Abstract
Multi-modal fusion and multi-model ensembling are prevalent in enhancing the performance of 3D semantic segmentation. Despite the impressive performance, these methods either rely on auxiliary input signals or suffer from costly computational expense. To efficaciously enhance the segmentation performance without introducing intolerable costs, we propose to transfer the rich knowledge from the multi-modal model (i.e., point clouds and images) and multiple model experts to the point-cloudbased network through knowledge distillation. Specifically, we present Information-oriented Heterogeneous Distillation (IHD) to help the uni-modal model absorb the complementary knowledge from the multi-modal teacher. We design the Information-Oriented Filtering (IOF) strategy to select informative images from the continuous image sequence for multi-modal fusion. This practice can boost the performance of the multi-modal teacher, thus benefiting the learning of the student. Besides, as opposed to vanilla model ensembling that requires the separate training of each expert, we propose Adept Snapshot Distillation (ASD). ASD treats the freely available model snapshots generated during the training phase as multiple experts, which significantly reduces the training cost for model ensembling. For each expert teacher, it only provides supervision to the student in the class where it is adept. The resulting Heterogeneous and Adept Snapshot Knowledge Distillation, dubbed HAS-KD, attains state-of-the-art results on ScanNetV2 and S3DIS datasets. HAS-KD can be seamlessly integrated into contemporary 3D segmentation algorithms and bring considerable gains without introducing extra inference burdens. The code will be made publicly available upon publication.

## 2. 方法论：核心思想、关键技术细节

**核心思想**：利用知识蒸馏，分别从两个维度提升学生模型：  
（1）**异构蒸馏（IHD）**：将多模态（点云+图像）教师的知识传递给纯点云学生。  
（2）**快照蒸馏（ASD）**：将训练过程中产生的多个模型检查点视为专家，按类别进行蒸馏。

### 关键技术细节

#### a) 信息导向异构蒸馏（IHD）
- **信息导向过滤（IOF）**：针对每个物体，从连续图像序列中挑选“语义丰度最高”的视图（即物体可见点比例最大），替代传统随机选取。  
  公式：语义丰度 \(A_j^i = M/N\)，其中 \(M\) 为物体 \(i\) 在视图 \(j\) 上的可见点数量，\(N\) 为物体总点数。  
- **蒸馏方式**：使用MSE损失，让学生特征（\(F_{\text{student}}\)）逼近多模态教师融合特征（\(F_{\text{fusion}}\)）：
  \[
  \mathcal{L}_{\text{IHD}} = \mathbb{E}[\|F_{\text{student}} - F_{\text{fusion}}\|^2]
  \]

#### b) 擅长快照蒸馏（ASD）
- **专家教师生成**：训练单一模型，每轮保存每个类别上表现最好的检查点（即每个类别的“专家”）。无需额外训练多个模型。  
- **蒸馏方式**：对于每个类别 \(i\)，仅使用该类别对应的专家教师 \(E_i\) 的特征 \(F_i\)，并基于类别掩码 \(M_i\) 选取对应体素的特征，构成集成特征 \(F_{\text{ensemble}}\)。学生特征受MSE损失监督：
  \[
  \mathcal{L}_{\text{ASD}} = \mathbb{E}[\|F_{\text{student}} - F_{\text{ensemble}}\|^2]
  \]
- **最终训练目标**：将IHD和ASD联合（ASD中教师换为IHD构建的多模态教师），总损失：
  \[
  \mathcal{L} = \mathcal{L}_{\text{Seg}} + \alpha \mathcal{L}_{\text{ASD-IHD}}
  \]
  默认 \(\alpha=4\)，\(\mathcal{L}_{\text{Seg}}\) 为交叉熵+Lovasz-softmax损失。

## 6. 主要结论与发现

- HAS-KD在ScanNetV2上达到78.6 mIoU，S3DIS Area 5上72.1 mIoU，均超越所有不用多模态/多数据集的先前方法。
- IHD和ASD各自都能提升基线（分别+0.4和+0.6 mIoU），联合使用提升更明显（+1.1 mIoU）。
- IOF相比随机过滤显著提升多模态教师性能（78.8 vs 78.1），并蒸馏出更好的学生。
- 专家集成（按类别选最佳检查点）优于平均集成（学生78.0 vs 77.6）。
- HAS-KD在推理时与基线PTV3完全相同的延迟和内存（61ms, 5.2GB），实现了“免费午餐”。

## 7. 优点

1. **思路新颖**：利用训练过程中自然产生的快照作为多专家，避免额外训练，非常实用。
2. **IOF策略**：基于语义丰度选择最佳视图，充分利用时序信息，提升多模态融合效果。
3. **按类别蒸馏**：避免弱小教师的噪声影响，提高蒸馏质量。
4. **高效性**：不改变学生架构，推理成本完全继承基线，性能提升显著。
5. **可集成性**：声明可无缝集成到当前3D分割算法中，具有通用潜力。

## 8. 不足与局限

1. **依赖实例标签**：IOF需要物体级别的实例掩码（可用离线模型生成，但额外依赖）。
2. **训练时长较长**：S3DIS上预训练3000 epoch，蒸馏1200 epoch，迭代成本较高。
3. **实验覆盖有限**：只在两个室内数据集上验证，未扩展到室外场景（如SemanticKITTI、nuScenes）或多种学生模型（如KPConv、稀疏卷积等）。
4. **类别专家获取需完整训练**：需训练完整模型并保存所有检查点，相当于一次全训练，对显存和存储有要求。
5. **公平性潜在偏差**：学生模型与教师模型结构相同（均为PTV3），未验证不同架构间的蒸馏效果。
