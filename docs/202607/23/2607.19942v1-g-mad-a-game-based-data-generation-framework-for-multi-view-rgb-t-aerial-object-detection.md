## 摘要
本文介绍了G-MAD，一个基于Arma3的开源框架，用于生成同步的多视角RGB-T航空目标检测数据。G-MAD解决了真实世界航空数据集构建的关键限制，包括有限的视角控制、不完美的RGB-T对齐以及高昂的标注成本。该框架支持结构化场景规范、可控的多视角相机布置、同步可见光/热成像捕获，以及利用引擎级几何元数据进行自动边界框标注。这些能力使得在航空目标检测中能够进行视角变化、多模态融合以及合成到真实迁移的受控研究。此外，利用G-MAD，我们构建并发布了AMOD，一个全新的大规模多视角航空RGB-T目标检测基准。源代码和数据集可在 https://unique-chan.github.io/G-MAD-Project 获取。

## Abstract
This work introduces G-MAD, an open-source framework that uses Arma3 to generate synchronized multi-view RGB-T data for aerial object detection. G-MAD addresses key limitations of real-world aerial dataset construction, including limited viewpoint control, imperfect RGB-T alignment and high annotation cost. The framework supports structured scenario specification, controllable multi-view camera placement, simultaneous visible/thermal capture, and automatic bounding box annotation using engine-level geometric metadata. These capabilities enable controlled studies of viewpoint variation, multi-modal fusion, and synthetic-to-real transfer in aerial object detection. Besides, using G-MAD, we construct and release AMOD, a new large-scale multi-view aerial RGB-T object detection benchmark. The source code and the dataset are available at https://unique-chan.github.io/G-MAD-Project.

  1. **视角控制有限**：难以在同一场景下获取多角度同步观测数据；
  2. **RGB-T对齐不完美**：真实多模态传感器融合存在配准误差和异源时间差；

- **整体含义**：本文提出G-MAD开源框架，将Arma3改造为可控、可扩展的合成数据生成流水线，并基于其生成多视角RGB-T航空目标检测基准AMOD，从而支持视角鲁棒性、多模态融合和合成到真实迁移的研究。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 框架架构（G-MAD）
- **三个阶段**：

  2. **场景采样与同步多视角多模态捕获**：每个场景下，通过可配置的视角采样模型（如“空对空”或“空对地”）生成多个视角的RGB-T同步图像。默认提供两种运动模式：“空对空”（横向扫掠）和“空对地”（抛物线轨迹倾斜视图）。视角采样器为模块化可扩展设计。

## 7. 优点

- **框架设计亮点**：
  - **高度可控**：用户仅需提供高层约束即可批量生成多样化场景，支持多地图、多天气、多时间、自定义视角采样轨迹。
  - **自动高质量标注**：利用引擎原生几何数据提取OBB/HBB，自动过滤遮挡，无需人工介入，且支持超采样（分辨率可配）。
  - **模块化与可扩展**：相机视角采样器、资产列表、场景生成逻辑均可自定义，便于扩展民用资产。
  - **GUI无关执行**：通过SQF脚本自动运行，支持多任务并发，不占用工作站显卡资源进行渲染。
- **数据集价值**：
  - AMOD是目前**最大规模**的多视角同步RGB-T航空检测数据集（73,920帧），且提供OBB标注；
  - 12类军事类别覆盖广泛，且包含低类间/高类内方差挑战（如Tank vs Armored、RADAR种类繁多）；
  - 采用6个同步视角，可系统研究视角退化影响，与仅提供离线多角度采样的先前工作不同（如DroneVehicle/DVTOD只提供独立视角，未保证同时性）；
  - 源代码和数据集已开放获取，可复现。

## 8. 不足与局限

- **实验覆盖局限**：
  - 仅采用单一检测器（Oriented R-CNN），未验证在其他架构（如YOLOv8、DETR-based）上的泛化性；
  - 合成到真实迁移仅与ImageNet预训练对比，缺乏与在真实RGB-T数据（如FLIR_ADAS、DroneVehicle）上预训练的基线对比；
  - 未定量评估G-MAD生成数据的保真度（如FID、LPIPS等图像质量指标）。
- **数据集偏差风险**：
  - 资产全部来自Arma3军事模型，类别偏重地面/空中战斗装备，缺乏民用目标（如行人、车辆、建筑），限制了民用应用；
  - 地图场景仍为游戏内置，与真实卫星/无人机照片存在域差距（纹理、阴影、背景复杂度）。
- **应用限制**：
  - 依赖Arma3商业游戏（需用户自行购买许可证），非完全独立的学术工具；
  - 仅支持非商业学术研究用途（依据EULA），商业落地需采用专业仿真平台（如VBS4）；
  - 热红外数据基于Arma3的伪热渲染引擎，并非真实物理模型（如MWIR光谱差异），需额外域迁移（论文提及可用图像翻译方法转换）。
- **算法创新性有限**：G-MAD本身属于系统工程贡献，核心标注方法基于引擎API调用，无新算法理论突破；SBS增强虽简单有效，但缺乏理论分析（如为何随机模糊边界比固定模糊更优）。
