## 摘要
基于摄像头的三维目标检测与跟踪是自动驾驶的核心任务，但在推理时缺乏昂贵且具有丰富深度信息的在线 LiDAR 时，精确的三维目标定位受到深度模糊性的根本限制。然而，在许多部署场景中，车辆会多次经过相同的环境，使得来自先前遍历生成的静态点云地图成为一种可行的几何先验来源。我们提出 DualViewMapDet，这是一种仅使用摄像头进行推理的框架，可在线检索此类地图先验，并利用它们来缓解部署过程中缺乏 LiDAR 传感器的问题。其关键理念是一种双空间的摄像头-地图融合策略，避免了单向视图转换。具体而言，我们 (i) 将地图投影到视角图像空间（Perspective View，PV），并编码多通道几何线索以丰富图像特征并支持鸟瞰图（BEV）提升；(ii) 直接在鸟瞰图（BEV）中对地图进行编码，使用稀疏体素骨干网络，并在共享度量空间中与来自摄像头的提升特征进行融合。在 nuScenes 和 Argoverse 2 上的大量评估表明，相较于强大的仅摄像头基线，我们的方法在性能上保持稳定提升，特别是在目标定位方面表现突出。消融实验进一步验证了 PV/BEV 融合和先验地图覆盖的贡献。我们已在 https://dualviewmapdet.cs.uni-freiburg.de 上发布代码与预训练模型。

## Abstract
Camera-based 3D object detection and tracking are central to autonomous driving, yet precise 3D object localization remains fundamentally constrained by depth ambiguity when no expensive, depth-rich online LiDAR is available at inference. In many deployments, however, vehicles repeatedly traverse the same environments, making static point cloud maps from prior traversals a practical source of geometric priors. We propose DualViewMapDet, a camera-only inference framework that retrieves such map priors online and leverages them to mitigate the absence of a LiDAR sensor during deployment. The key idea is a dual-space camera-map fusion strategy that avoids one-sided view conversion. Specifically, we (i) project the map into perspective view (PV) and encode multi-channel geometric cues to enrich image features and support BEV lifting, and (ii) encode the map directly in bird's-eye view (BEV) with a sparse voxel backbone and fuse it with lifted camera features in a shared metric space. Extensive evaluations on nuScenes and Argoverse 2 demonstrate consistent improvements over strong camera-only baselines, with particularly strong gains in object localization. Ablations further validate the contributions of PV/BEV fusion and prior-map coverage. We make the code and pre-trained models available at https://dualviewmapdet.cs.uni-freiburg.de .

## 二、方法论概述（DualViewMapDet）

### 1. 核心思想

- 提出 **DualViewMapDet**：一种双视图空间（Perspective View + Bird’s-Eye View）融合策略，使相机与地图的几何信息在两个空间中互补。  
- **关键理念**：  
  （1）在透视视图（PV）下，地图投影能够辅助相机特征学习深度；  
  （2）在俯视视图（BEV）下，稀疏体素编码的地图提供真实空间尺度的几何结构，有助于3D空间定位。  
- **优势**：避免单一视域下的特征错配问题，实现跨表示空间的几何约束对齐。

### 2. 算法流程与结构

1. **输入与检索**
   - 输入：多视角 RGB 图像、相机内外参、当前车辆位姿。  
   - 检索：从预构建的全局点云地图中按当前位置提取局部地图块（约 R 范围内）。  

2. **双空间特征编码**
   - **PV分支**：  
     - 将地图点投影至每张图像平面，得到稀疏深度图及有效性掩码。  
     - 构建多通道 PV 几何编码：包括深度值、位置嵌入（positional embedding）、三维坐标 (x,y,z)、最近有效深度扩散、距离有效像素的归一化距离等。  
     - 与相机图像特征拼接，形成融合的 PV 表征。  
   - **BEV分支**：  
     - 将检索到的地图体素化（voxel size 0.2m），由稀疏 3D 卷积网络提取 BEV 地图特征。  

3. **跨空间融合**
   - **PV融合**：PV地图特征与图像特征串接，经卷积整合。  
   - **BEV提升与融合**：  
     - 利用 LSS（Lift-Splat-Shoot）模块将 PV 融合特征提升至 BEV 格栅。  
     - 与 BEV 地图特征再次融合（3×3卷积），并通过轻量FPN结构整合上下文信息。  

4. **检测与跟踪解码**
   - 采用基于 Sparse4Dv3 改进的 Transformer 检测头：  
     - 对每个目标 Query 执行 **BEV-PV两次可变形聚合**（Deformable Aggregation），先聚合度量几何上下文（BEV），再精化语义细节（PV）。  
     - 输出3D框中心、尺寸、朝向、速度等。  

5. **训练与增强**
   - 损失函数：  
     \[
     L_{total} = \lambda_{det} L_{det} + \lambda_{depth} L_{depth}
     \]
     - \(L_{det}\)：检测与回归损失；  
     - \(L_{depth}\)：LSS深度预测监督（训练时利用LiDAR深度真值，测试时不使用）。  
   - **图像–地图网格遮罩（Grid Masking）**：  
     训练时在图像与地图空间执行网格遮挡，以防模型过度依赖地图先验、提升稳健性。

## 七、方法优点与创新点

- **创新的双空间融合架构**：同时在 PV 与 BEV 两个空间中对齐图像与地图特征，解决以往仅PV或仅BEV融合的表达失配问题。  
- **多通道 PV 几何编码**：细致建模深度置信度与空间上下文，提高稀疏地图的有效利用率。  
- **网格遮罩增强策略**：提升模型对不完整地图先验的鲁棒性，防止过拟合静态几何。  
- **仅摄像头推理，无需实时LiDAR**：在部署阶段成本低、兼容性高。  
- **系统化消融分析**：从融合位置、特征构成、地图覆盖到稳健性均有严格实验支持。
