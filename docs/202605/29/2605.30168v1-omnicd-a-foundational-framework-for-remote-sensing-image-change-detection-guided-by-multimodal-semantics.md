## 摘要
遥感中的变化检测对于城市监测和灾害评估等应用至关重要，但传统方法难以在不同场景下泛化。我们提出了OmniCD，一个通过多模态语义引导统一并增强遥感变化检测的基础框架。OmniCD将图像和文本提示（如文本描述、语义地图和地理空间元数据）整合到一个统一架构中，支持从二值变化检测到零样本语义变化理解的任务。该框架集成了层次化场景检索模块和变化检测模块，并通过风格解耦机制增强跨域鲁棒性。我们还引入了RSITCD，一个包含30万以上标注图像-文本对的大规模多模态数据集。大量实验表明，OmniCD在多个基准上达到了最先进的性能，展现出强大的适应性，并为遥感通用变化检测系统奠定了坚实基础。

## Abstract
Change detection (CD) in remote sensing is vital for applications such as urban monitoring and disaster assessment, yet traditional methods struggle with generalization across diverse scenarios. We present OmniCD, a foundational framework that unifies and enhances remote sensing CD through multimodal semantic guidance. OmniCD incorporates image and text prompts -- such as textual descriptions, semantic maps, and geospatial metadata -- into a unified architecture, supporting tasks from binary CD to zero-shot semantic change understanding. The framework integrates a hierarchical scene retrieval module and a change detection module, reinforced by a style disentanglement mechanism for improved cross-domain robustness. We further introduce RSITCD, a large-scale multimodal dataset with 300K+ annotated image-text pairs. Extensive experiments show that OmniCD achieves state-of-the-art performance across benchmarks, demonstrating strong adaptability and setting a solid foundation for general-purpose CD systems in remote sensing.

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：提出“引导器-检测器”协作架构（Guider–Detector Architecture），利用多模态提示（文本描述、参考图像）生成感兴趣区域（ROI）注意力图，引导检测器重点关注与提示语义相关的变化，同时引入风格解耦机制抑制成像条件导致的伪变化。
- **关键技术细节**：
  - **特征提取模块**：
    - 图像编码器：使用MAE预训练的ViT-H/16（14×14窗口注意力+4个全局注意力块），输入512×512，输出64×64特征图，经1×1和3×3卷积降维至256通道。
    - 文本编码器：使用BERT-base，输出经平均池化得到256维文本嵌入。
  - **引导模块（基于SAM的Transformer解码器）**：
    - 输入：图像嵌入（64×64×256）、文本/参考图像提示嵌入。
    - 包含两层解码器，每层执行：自注意力（提示令牌间）→ 交叉注意力（提示令牌查询图像嵌入）→ MLP更新提示令牌 → 交叉注意力（图像嵌入查询提示令牌）。
    - 输出：经上采样（4×转置卷积）后的图像嵌入与输出令牌相乘，生成单图像掩码，再对两时相掩码逐像素取最大值得到ROI注意力图。
    - 嵌入维度256，注意力头数8，交叉注意力中QKV通道减半至128以提高效率。
  - **检测模块**：
    - 输入：两时相图像特征差值（绝对值的通道级差）。
    - 使用金字塔场景解析（PSP）模块，在不同尺度（1×1、2×2、3×3、6×6）上池化并融合多尺度上下文，再经卷积层生成初始变化图。
    - 最后将初始变化图与引导模块输出的注意力图逐点相乘，仅保留感兴趣区域的变化。
  - **风格解耦模块**：
    - 目的：分离内容特征（语义信息）与风格特征（成像条件如传感器、光照、大气），抑制伪变化。
    - 实现：在特征提取模块上增加孪生风格分支，通过轻量级编码器（逐步下采样至单值）提取全局风格特征。
    - 通过AdaIN（自适应实例归一化）在解码器中融合内容与风格特征，重建输入图像，确保信息不丢失。引入特征分离损失、内容相似性损失（未变化区域）、图像重建损失（L1）等多项损失函数联合优化。
  - **损失函数**：多任务损失 = 变化检测损失（BCE + Dice） + 特征分离损失（惩罚内容与风格相关性） + 内容相似性损失（未变化区域特征分布一致） + 图像重建损失（L1）。
- **推理流程**：输入双时相图像+文本提示（或参考图像+掩码）→特征提取→引导模块生成ROI注意力→检测模块生成变化图并与注意力图融合→输出最终变化检测结果。支持文本提示和参考图像提示两种方式，无需额外训练即可零样本迁移。

## 3. 实验设计：数据集、benchmark、对比方法
- **数据集**：
  - 训练：自建大规模多模态数据集RSITCD（152,123个样本，即304,246张图像），整合21个公开CD数据集，覆盖17类地物、40+种变化类型，空间分辨率0.03–2m，所有图像标准化为512×512。
  - 测试：
