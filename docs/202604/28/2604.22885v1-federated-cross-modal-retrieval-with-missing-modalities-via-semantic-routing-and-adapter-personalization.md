## 摘要
联邦跨模态检索在异构客户端数据下面临严峻挑战，特别是非独立同分布（non-IID）的语义分布和模态缺失。在这种异质性条件下，单一的全局模型往往难以同时捕获共享的跨模态知识和客户端特定特征。我们提出了 RCSR，一种支持个性化的联邦学习框架，结合了原型锚定（prototype anchoring）、面向检索的语义路由（retrieval-centric semantic routing）以及可选的客户端特定适配器。RCSR 构建于冻结的 CLIP 主干网络之上，利用轻量级共享适配器实现全局知识传递，同时支持高效的本地个性化。原型锚定帮助单模态客户端与全局跨模态语义对齐，而服务器端的语义路由器则基于检索一致性自适应地分配聚合权重，从而在异构更新中减轻对齐漂移。大量在 MS-COCO、Flickr30K 及其他基准数据集上的实验表明，RCSR 显著提升了全局检索精度与训练稳定性，并进一步增强了客户端层面的检索性能，尤其是对于缺失模态的客户端。代码可在 https://github.com/RezinChow/RCSR-Retrieval-Centric-Semantic-Routing 获取。

## Abstract
Federated cross-modal retrieval faces severe challenges from heterogeneous client data, particularly non-IID semantic distributions and missing modalities. Under such heterogeneity, a single global model is often insufficient to capture both shared cross-modal knowledge and client-specific characteristics. We propose RCSR, a personalization-friendly federated framework that integrates prototype anchoring, retrieval-centric semantic routing, and optional client-specific adapters. Built on a frozen CLIP backbone, RCSR leverages lightweight shared adapters for global knowledge transfer while supporting efficient local personalization. Prototype anchoring helps unimodal clients align with global cross-modal semantics, and a server-side semantic router adaptively assigns aggregation weights based on retrieval consistency to mitigate alignment drift during heterogeneous updates. Extensive experiments on MS-COCO, Flickr30K, and other benchmarks show that RCSR consistently improves global retrieval accuracy and training stability, while further enhancing client-level retrieval performance, especially for clients with incomplete modalities. Code is available at https://github.com/RezinChow/RCSR-Retrieval-Centric-Semantic-Routing.

## 一、核心问题与研究背景

- **研究动机**  
  联邦学习在多组织或多设备协作中能够确保隐私，但在跨模态检索任务中（如图文检索），存在两个主要挑战：
  1. **非IID数据分布**（不同客户端数据语义差异大）导致共享模型难以保持一致的跨模态对齐。  
  2. **模态缺失问题**，即部分客户端仅有图像或仅有文本，使得标准的对比学习无法进行。  

- **总体目标**  
  建立一个在联邦场景中可同时处理数据异质性与模态缺失的高效检索框架，既维持全局跨模态语义一致性，又保留客户端的个性化特征，同时降低通信及隐私风险。

## 二、方法论与技术框架

### 1. 整体思路
提出 **RCSR（Retrieval-Centric Semantic Routing）** 框架，核心思想是：
- 将跨模态检索视为**检索一致性最优**问题，而不仅是参数平均；
- 引入语义路由机制和个性化适配器，使得聚合过程考虑客户端的语义影响与对齐质量；
- 在冻结的 **CLIP ViT-B/32** 骨干上，实现**轻量化联邦优化**。

### 2. 关键组成模块
1. **原型锚定（Prototype Anchoring）**  
   - 单模态客户端无法计算图文对比损失，因此引入全局原型（图像/文本均值向量）进行锚定。  
   - 使用余弦相似度最小化局部表示与缺失模态的全局原型差异。  
   - 公式：  
     \( L_{anchor} = 1 - \frac{1}{B} \sum_i \cos(z_i, \bar{p}_{\neg m_k}) \)

2. **语义路由（Semantic Router）**  
   - 路由器根据每个客户端的统计特征（原型、几何信息、损失等）输出**聚合权重** \( w_R \)。  
   - 架构：两层 MLP + Transformer 自注意力层，带模态掩码处理缺失输入。  
   - 目标：最小化加权原型与全局原型的余弦差，并保持多样性（熵正则）。

3. **鲁棒公平聚合（Robust-Fair Aggregation）**  
   - 通过**最小–最大公平约束**优化各客户端权重 \( q_k \)：  
     \(\min_\theta \max_{q \in \Delta^{N-1}} \sum_k q_k L_k(\theta) + \tau H(q)\)  
   - 最终融合权重： \( w_k = \frac{w_R^k q_k}{\sum_j w_R^j q_j} \)

4. **适配器个性化（Adapter Personalization）**  
   - 客户端维护轻量**适配器层 A_k**（两层 MLP），只在本地更新，不上传。  
   - 其调整强度由路由器权重逆向决定：权重低的客户端获得更强的个性化补偿。  
   - 避免全局模型受高偏客户端扰动。

### 3. 算法流程（简述）
1. 服务器广播全局模型和全局原型；
2. 客户端本地训练（有/无配对，根据模态类型使用不同Loss）；
3. 客户端上传统计向量；
4. 服务器通过路由器+公平优化计算聚合权重；
5. 更新全局模型与EMA原型；
6. 稳定后进入个性化阶段，单独微调本地适配器。

## 三、实验设计

## 七、优点与亮点

- **创新的语义路由机制**：用“检索一致性”指导聚合，比传统权重分配更符合跨模态任务性质。
- **原型锚定设计**：优雅地解决单模态客户端的训练断层问题。
- **轻量化架构**：仅传输 <2% 参数、通信效率高。
- **公平性与稳定性兼顾**：Minimax优化显著降低客户端性能差异。
- **个性化适配**：无需额外通信即可实现有针对性的本地微调。
- **理论支撑完整**：包含收敛性与鲁棒性分析，符合标准联邦优化框架。
